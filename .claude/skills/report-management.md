# Report Management Skill

Manage genomics analysis reports — create, update, list, and serve markdown reports.

## Report Location

All reports are stored at: `/data/app/reports/`

## Report Types

| Category | Naming | Generated By |
|----------|--------|-------------|
| **pgs** | `PGS{ID}.md` | Auto — on scoring run completion |
| **run** | `run_{ID}.md` | Auto — on scoring run completion |
| **custom** | Any `.md` | Manual — via this skill or API |
| **summary** | `summary_*.md` | Manual — aggregation reports |

## Creating Reports

### Quick Report
```bash
cat > /data/app/reports/my_report.md << 'EOF'
# Report Title

Content here...
EOF
```

### Via API
```bash
curl -X POST http://localhost:8600/genomics/api/reports/create \
  -H "Content-Type: application/json" \
  -d '{"filename": "my_report.md", "content": "# Title\n\nContent...", "category": "custom"}'
```

## Common Tasks

### List all reports
```bash
curl -s http://localhost:8600/genomics/api/reports/list | python3 -m json.tool
```

### View a report
```bash
curl -s http://localhost:8600/genomics/api/reports/content/PGS000001 | python3 -m json.tool
```

### Regenerate all run reports
```bash
curl -X POST http://localhost:8600/genomics/api/reports/regenerate-all
```

### Generate report for a specific run
```bash
curl -X POST http://localhost:8600/genomics/api/reports/generate/{run_id}
```

### Delete a report
```bash
curl -X DELETE http://localhost:8600/genomics/api/reports/content/old_report
```

## Report Template

When creating analysis reports, use this structure:

```markdown
# {Title}

> **Date**: {date}
> **Analyst**: Claude
> **Samples**: {sample list}

---

## Summary

Key findings...

## Results

| Sample | Score | Z-Score | Percentile | Risk |
|--------|-------|---------|------------|------|
| ...    | ...   | ...     | ...        | ...  |

## Interpretation

Clinical context...

## Methods

Scoring method, reference panels, etc.

---

*Report generated {date}*
```

## Viewing Reports

- **Dashboard**: Reports tab in the genomics dashboard
- **Mini-site**: `/genomics/reports/` — standalone viewer with sidebar navigation
- **Raw markdown**: `/genomics/api/reports/raw/{filename}`
- **API JSON**: `/genomics/api/reports/content/{filename}`

## Auto-Generation

Every completed scoring run automatically generates:
1. **Per-run report** (`run_{id}.md`) — full summary of all PGS scored in that run
2. **Per-PGS reports** (`PGS{id}.md`) — aggregated across all runs for that PGS

To ensure reports are always up-to-date, the checklist sync (`sync_checklist_from_db()`) runs after every scoring completion and regenerates per-PGS reports when new results are available.
